# Web UI 使用指南

## 概述

DevToolsLLM 提供了一个现代化的 Web 界面，用于实时监控和分析 LLM 应用的工具调用。

## 快速开始

### 启动步骤

1. **启动后端监控服务器**
   ```bash
   devtools monitor 3000
   ```

2. **启动前端开发服务器**
   ```bash
   cd /path/to/devtools-llm
   npm run dev:frontend
   ```

3. **访问 Web 界面**
   ```
   http://localhost:3001
   ```

## 功能详解

### 1. Dashboard（仪表盘）

#### 统计卡片

显示实时统计数据：

**总调用次数**
- 显示累计的工具调用总数
- 实时更新
- 点击可查看详细历史

**成功率**
- 成功调用占总调用的百分比
- 颜色编码：
  - 绿色（≥90%）：健康
  - 黄色（70-90%）：警告
  - 红色（<70%）：危险

**平均耗时**
- 所有工具调用的平均执行时间
- 单位：毫秒（ms）
- 帮助识别性能瓶颈

**活跃客户端**
- 当前连接到监控服务器的客户端数量
- 显示实时连接状态

#### 最近调用列表

显示最新的 10 次工具调用：

**显示信息：**
- ✅/❌ 状态图标
- 工具名称（高亮显示）
- 执行耗时
- 时间戳
- 错误信息（如果失败）

**交互功能：**
- 悬停显示完整信息
- 点击展开详细参数
- 错误调用红色高亮

#### 调用链可视化

ASCII 树形图展示工具调用关系：

```
├─ ✅ search (120ms)
├─ ✅ calculator (45ms)
└─ ✅ weather (80ms)
   └─ ❌ Error: API timeout
```

**特点：**
- 清晰的层级关系
- 状态图标
- 执行时间
- 错误节点特殊标记

### 2. Errors（错误分析）

**即将推出的功能：**

#### 错误趋势图
- 时间序列图表
- 显示错误发生频率
- 支持时间范围选择

#### 错误类型分布
- 饼图/柱状图
- 显示各类错误占比
- 可点击筛选

#### 智能调试建议
- 基于错误类型的建议
- 历史修复案例
- 相关文档链接

#### 错误详情面板
- 完整错误堆栈
- 相关调用上下文
- 参数信息
- 重现步骤

### 3. Traces（追踪查看器）

**即将推出的功能：**

#### 交互式调用链图
- 可缩放的图形界面
- 节点点击查看详情
- 支持搜索和过滤

#### 时间线视图
- 横向时间轴
- 显示调用时序
- 并发调用可视化

#### 性能分析
- 火焰图
- 瓶颈识别
- 优化建议

#### 调用详情面板
- 输入参数
- 输出结果
- 执行时间
- 调用栈

### 4. Settings（设置）

**即将推出的功能：**

#### 监控配置
- WebSocket 端口设置
- 数据刷新频率
- 历史数据保留时长

#### 告警规则
- 错误率阈值
- 性能阈值
- 通知方式（邮件、Slack）

#### 数据管理
- 导出历史数据
- 清空缓存
- 数据归档

#### 外观设置
- 主题切换（暗色/亮色）
- 字体大小
- 图表样式

## 实时更新机制

### WebSocket 连接

Web UI 通过 WebSocket 与后端保持持久连接：

```javascript
// 自动连接
const ws = new WebSocket('ws://localhost:3000');

// 接收实时数据
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  
  switch(message.type) {
    case 'call':
      // 新的工具调用
      updateDashboard(message.data);
      break;
    case 'history':
      // 历史数据
      loadHistory(message.data);
      break;
  }
};
```

### 连接状态指示器

侧边栏底部显示连接状态：

- **🟢 Connected** - 已连接，实时更新
- **🟡 Connecting** - 正在连接
- **🔴 Disconnected** - 连接断开，尝试重连

### 自动重连

连接断开时自动尝试重连：
- 初始延迟：1 秒
- 最大延迟：30 秒
- 指数退避策略

## 故障排查

### 无法连接到服务器

**问题：** 显示 "Disconnected"

**解决方案：**
1. 确认后端服务器正在运行：`devtools monitor 3000`
2. 检查端口是否被占用
3. 查看浏览器控制台错误信息
4. 检查防火墙设置

### 数据不更新

**问题：** 统计数据不变化

**解决方案：**
1. 检查 WebSocket 连接状态
2. 刷新页面
3. 清除浏览器缓存
4. 重启后端服务器

### 页面加载缓慢

**问题：** 首次加载很慢

**解决方案：**
1. 检查网络连接
2. 清理历史数据
3. 减少数据保留时长
4. 使用生产构建：`npm run build:frontend`

## 高级用法

### 自定义主题

编辑 `client/index.css`：

```css
:root {
  --primary-color: #4CAF50;
  --background-color: #0f0f0f;
  --text-color: #e0e0e0;
  --border-color: #333;
}
```

### 集成到现有项目

将 Web UI 嵌入到你的应用：

```html
<iframe 
  src="http://localhost:3001" 
  width="100%" 
  height="600px"
  frameborder="0">
</iframe>
```

### API 调用

直接调用后端 API：

```javascript
// 获取历史数据
fetch('http://localhost:3000/api/history')
  .then(res => res.json())
  .then(data => console.log(data));

// 获取统计数据
fetch('http://localhost:3000/api/stats')
  .then(res => res.json())
  .then(stats => console.log(stats));
```
